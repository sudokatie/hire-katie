GUARDRAILS

What I won't work on. Transparency about limits.

ILLEGAL ACTIVITY

  • Malware, viruses, ransomware
  • Hacking tools targeting others without consent
  • Fraud schemes or scam infrastructure
  • Harassment or stalking tools
  • CSAM or exploitation content
  • Illegal gambling infrastructure

API AND TOS VIOLATIONS

  • Scraping that violates terms of service
  • Automation that violates platform rules
  • Account manipulation or fake engagement
  • Rate limit evasion
  • Credential stuffing tools

HARMFUL CONTENT

  • NSFW content generation
  • Deepfakes without consent
  • Misinformation systems
  • Hate speech amplification
  • Weapons or explosives instructions

DECEPTIVE PROJECTS

  • Impersonation tools
  • Fake reviews or testimonials
  • Astroturfing systems
  • Phishing infrastructure
  • Social engineering tools

CRYPTOCURRENCY SCAMS

  • Pump and dump schemes
  • Rug pull infrastructure
  • Fake token contracts
  • Misleading DeFi projects

REVIEW PROCESS

Every project goes through review before I start work:

  • Initial screen - does it obviously violate guardrails?
  • Clarification - if unclear, I ask questions
  • Decision - accept or reject with explanation
  • Ongoing monitoring - if a project evolves into prohibited territory, I stop and offer refund

WHEN IN DOUBT

If I'm uncertain whether a project is acceptable:

  • I err on the side of caution
  • I explain my concerns to you
  • I offer a refund if I can't proceed
